Progressive Enchancement

Wat is Progressive Enchancement?

Progressive Enhancement is een strategie in web design die de nadruk legt op web content first. Dit houdt in dat de content wordt gescheiden van de semantiek van de presentatie. Het uiteindelijk doel van Progressive Enhancement is dat de basis content van een website/applicatie voor iedereen beschikbaar is, gebruikers die een browser hebben met features krijgen de enhanced versie van een website te zien.

In een simpele beeldvorming is Progressive Enchancement de scheiding van HTML, CSS en Javascript zoals onderstaande afbeelding:

Best Practices

  • Vermijd Inline Styling Om ervoor te zorgen dat je HTML semantisch is is het belangrijk om inline styling te vermijden. Hierdoor kun je gemakkelijker een style veranderen als iemand op een andere browser of device zit.
  • Vermijd Inline Javascript Zorg er ook voor dat het JavaScript gedrag niet in de HTML wordt gedefinieerd, maar alleen in het javascript bestand. Het is bijvoorbeeld beter om een onclick element niet op het HTML element toe te voegen, maar door deze te selecteren via JavaScript d.m.v. een querySelector en vervolgens een eventlistener daaraan toe te voegen.
  • Gebruik een Systeemfont Als een gebruiker op een erg sloom netwerk zit, kan het soms de experience zodanig verslomen als je de fonts inlaad als webfont. Het is daarom verstandig om eerst een systeemfont in te stellen, wanneer de content is geladen laad je dan het webfont in. 
                            
body {
   /* Use fallback font */
   font-family: sans-serif;
}

/* Add this class to the html via 
   JavaScript if the font 'Open Sans' exists. */
   
.enhanced-font body {
   font-family: Open Sans;
}
    Gebruik hiervoor bijvoorbeeld fontFaceObserver
  • Mobile First Door een mobile-first benadering aan te gaan zorg je ervoor dat er zoveel mogelijk regels css worden hergebruikt. Dat betekent dus dat er minder regels css zijn, dus ook minder voor de gebruiker om in te laden.
  • Lazy Loading Lazy loading is een manier om afbeeldingen pas in te laden zodra deze in de viewport komen van de gebruiker. Helaas wordt hiervoor ook JavaScript gebruikt en heb je dus een manier nodig om de afbeeldingen als nog te tonen aan mensen die geen JavaScript gebruiken. 
                            
<img data-src="path/to/image.jpg" loading="lazy" />

<noscript>
<img  src="path/to/image.jpg" />
</noscript>
